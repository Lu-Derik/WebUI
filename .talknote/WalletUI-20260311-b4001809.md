## 2026-03-11T03:06:23.589Z

### User

1）如何在VS Code的插件开发项目中直接试用本项目的插件？
2）如何在插件开发项目中启动 ”Extension Development Host“？

### Copilot

1) 在 VS Code 中直接试用本扩展
- 在 VS Code 中打开扩展项目的根目录（包含 package.json）。
- 如果是 TypeScript，先安装/编译：  
```bash
npm install
npm run compile
```
- 按 F5 或 在侧边栏选择“运行和调试”然后点击“运行”启动 Extension Development Host（新窗口会以 Extension Development Host 运行并加载当前扩展）。  
- 在新窗口中测试命令/面板/激活事件。

2) 启动 Extension Development Host（可选项/配置）
- 快捷：F5（启动并附带调试）或 Ctrl+F5（不带调试）。  
- 或在“运行和调试”面板选择 “Launch Extension”/“Run Extension” 配置后点击播放按钮。  
- 确认 .vscode/launch.json（示例）：
```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Run Extension",
      "type": "extensionHost",
      "request": "launch",
      "runtimeExecutable": "${execPath}",
      "args": ["--extensionDevelopmentPath=${workspaceFolder}"],
      "outFiles": ["${workspaceFolder}/out/**/*.js"]
    }
  ]
}
```

如需在已打开的主窗口外再开一个宿主窗口，使用 F5 会自动打开名为 “Extension Development Host” 的新窗口。

- 先编译/安装（若 TypeScript）

- 注意扩展不会自动“看起来启动”直到触发 activation event：  
  - 如果 activationEvents 是 onCommand，用 Extension Development Host 窗口按 Cmd+Shift+P，运行你 package.json 中 contributes.commands 的命令。  
  - 如果是 onLanguage/onStartup，把相应文件/视图打开以触发。

  - 在主 VS Code 窗口打开 View → Output，右上选择 “Log (Extension Host)” 或 “Extension Host” 查看输出。

- 如果使用 source-map/attach：确认 outFiles 与编译输出路径匹配，或启用 tsc --watch 以确保最新 JS 可被加载。

按照上面顺序检查，一般能定位是编译、launch.json、还是 activation 未触发导致“看不到扩展运行”的问题。

下面是精简的排查步骤，逐项检查即可：

- 确认选择了正确的调试配置（Run/Debug 面板的下拉框选中你写的 “Run Extension”/“Launch Extension”），再点绿色播放按钮或 Run → Start Debugging。  
- 如果是 TypeScript，先编译或开 watch：
```bash
npm install
npm run compile      # 或 npm run watch
```
- 确认 launch.json 指向扩展目录并指定 outFiles：
```json
{
  "name": "Run Extension",
  "type": "extensionHost",
  "request": "launch",
  "runtimeExecutable": "${execPath}",
  "args": ["--extensionDevelopmentPath=${workspaceFolder}"],
  "outFiles": ["${workspaceFolder}/out/**/*.js"]
}
```
- 激活事件未触发时扩展看起来“没跑”：检查 package.json 的 activationEvents。若是 onCommand，需在 Extension Development Host 窗口按 Cmd+Shift+P 手动运行你注册的命令；若是 onLanguage/onStartup，打开对应文件或触发相应事件。  
- 查看日志与错误：  
  - 在主窗口打开 View → Output，右侧下拉选择 “Log (Extension Host)” 或 “Extension Host” 查看日志。  
  - 在 Extension Development Host 窗口用 Help → Toggle Developer Tools 查看 console（console.log /异常会在这里）。  
- 在 Extension Development Host 的 Extensions 侧栏确认你的扩展以 Dev 标记存在且已启用。  
- 多根工作区或目录不对时，修改 args 为具体路径：
```json
"args": ["--extensionDevelopmentPath=/绝对/路径/到/扩展文件夹"]
```
- 如果仍无效，检查 Debug Console / DevTools 是否有加载/解析 outFiles、source-map 或语法错误信息，根据错误修复后重启调试。

按上述顺序检查一般能定位到“没启动”的原因。
